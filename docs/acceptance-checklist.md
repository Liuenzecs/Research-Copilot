# Research Copilot 验收清单

本文档用于当前版本的手工验收，重点面向：

- 单人研究者的真实周使用节奏
- 论文工作区 -> 复现工作区 -> 心得 -> 周报 -> Memory 的闭环
- 功能是否可用
- 上下文是否连贯
- UI / 文案是否自然

---

## 建议验收顺序

建议按下面顺序验收，最接近真实使用链路：

1. 顶栏导航与功能说明
2. 论文搜索与阅读页
3. 论文到复现闭环
4. 复现执行与日志阻塞
5. 心得页
6. 周报工作区
7. Memory
8. 仪表盘与全局状态

---

## A. 论文搜索与阅读页

### 0. 顶栏与帮助抽屉

- [ ] 左侧边栏不再出现
- [ ] 右侧上下文面板不再出现
- [ ] 顶栏能看到主要导航入口
- [ ] 点击“功能说明”后，会打开右侧抽屉
- [ ] 抽屉中能看到：
  - [ ] 产品定位
  - [ ] 推荐工作流
  - [ ] 主要页面说明
  - [ ] 翻译说明
  - [ ] 本地优先说明

### 0.1 设置页运行路径

- [ ] 打开 `/settings` 时，能看到当前实际运行的：
  - [ ] 数据库 URL
  - [ ] 数据库文件路径
  - [ ] 数据目录
  - [ ] 向量目录
- [ ] 若本地存在历史 `backend/backend/data` 目录，会看到已自动收口的提示

### 1. 搜索与阅读拆分

- [ ] `/search` 只负责搜索，不再右半屏嵌入论文工作区
- [ ] 在 `/search` 搜索到论文后，点击结果会进入 `/papers/[paperId]`
- [ ] 访问旧深链 `/search?paper_id=<id>&summary_id=<id>` 时，会自动跳到新阅读页
- [ ] 进入新阅读页后，论文元信息、abstract、摘要、研究状态能正常显示

### 2. 正文阅读与 PDF 状态

- [ ] 未下载 PDF 时，会看到“下载 PDF 并生成阅读文本”入口
- [ ] 已下载 PDF 且正文可解析时，会看到本地 PDF 抽取后的正文段落
- [ ] 已下载 PDF 但正文暂不可读时，会出现非阻塞提示，仍可继续使用工作区
- [ ] 阅读页正文区域是舒适排版的全宽阅读面，而不是右半屏嵌入区域

### 3. 选词翻译

- [ ] 只在正文文本区域选中文本时，才会出现“翻译选中内容”按钮
- [ ] 点击后会返回中文翻译结果，并保留英文原文
- [ ] 翻译结果会显示免责声明“英文原文 canonical，中文仅辅助理解”
- [ ] 若公共翻译接口不可用，仍会返回本地辅助翻译而不是报 500

### 4. 摘要语义

- [ ] 摘要选择器切换后，页面展示内容跟随当前选中的摘要变化
- [ ] 页面不再总是显示“最新摘要”
- [ ] 默认会选中第一条摘要
- [ ] 选择“`不绑定摘要`”后，摘要区显示 paper-only 提示，而不是继续显示某条 summary

### 5. Reflection 绑定

- [ ] 选择具体摘要后，新建 paper reflection 会写入对应 `summary_id`
- [ ] 选择“`不绑定摘要`”后，新建 paper reflection 不会写入 `summary_id`
- [ ] 无摘要绑定时，reflection 预填会回退到 paper abstract / paper metadata 语义

### 6. 进入复现

- [ ] 点击“进入复现工作区 / 继续复现”后，跳转到 `/reproduction?paper_id=<id>`
- [ ] 即使论文还没下载 PDF、还没有摘要，也不阻塞进入复现页

## B. 论文到复现闭环

### 1. Paper Context 带入

- [ ] 进入 `/reproduction?paper_id=<id>` 后，页面能自动识别当前论文上下文
- [ ] 顶部能显示当前 paper title 与 `paper_id`

### 1.1 去 ID 化入口

- [ ] 打开 `/reproduction` 时，不再要求手工输入 `paper_id / reproduction_id`
- [ ] 可直接按论文标题或关键词搜索进入复现上下文
- [ ] 页面能看到“最近复现记录”列表，并可直接继续某一条记录

### 2. 历史 reproduction 续接

- [ ] 如果当前论文已有历史 reproduction，会自动打开最近一条
- [ ] 页面明确显示“继续最近一次复现”
- [ ] 页面仍保留“新建新的复现记录”入口

### 3. 新建 reproduction

- [ ] 如果当前论文没有历史 reproduction，页面会进入“准备新建复现”状态
- [ ] 自动 repo 搜索会触发
- [ ] 不选 repo 也能按 paper-only 成功创建 reproduction plan

### 4. Repo Candidates

- [ ] repo candidates 会按论文标题自动搜索
- [ ] 选中 repo 后能按 `paper_id + repo_id` 创建计划
- [ ] 不选 repo 时按 `paper_id` 创建计划仍成功
- [ ] repo 自动搜索失败时，只出现 warning，不阻塞新建流程

## E. 周报工作区

### 1. 默认加载

- [ ] 打开 `/dashboard/weekly-report` 时，会自动加载当前周 live context
- [ ] 同时会加载历史草稿列表

### 2. 生成草稿分流

- [ ] 当前周没有草稿时，点击“生成周报草稿”会直接创建新 draft
- [ ] 当前周已有草稿时，点击“生成周报草稿”会弹出选择框：
  - [ ] 继续最近草稿
  - [ ] 新建一份草稿
  - [ ] 取消

### 3. 继续最近草稿

- [ ] 点击“继续最近草稿”后：
  - [ ] 不重新生成 markdown
  - [ ] 右侧打开该周最近一条草稿
  - [ ] 左侧显示该 draft 保存时的 snapshot context
  - [ ] 页面提示“正在继续编辑该周最近草稿”

### 4. 新建一份草稿

- [ ] 点击“新建一份草稿”后：
  - [ ] 创建新的 draft
  - [ ] 右侧打开新 draft
  - [ ] 左侧保持 live context

### 5. 编辑器

- [ ] 编辑器顶部能显示：
  - [ ] 标题
  - [ ] 周范围
  - [ ] 中文状态
  - [ ] 最后更新时间
- [ ] 状态显示为中文：
  - [ ] 草稿
  - [ ] 已定稿
  - [ ] 已归档
- [ ] 修改 title / markdown / status 后，会出现“未保存修改”提示
- [ ] 点击“恢复到已保存版本”后，只回退本地内容，不写后端
- [ ] 点击保存后，草稿内容和更新时间会刷新

### 6. 历史草稿列表

- [ ] 历史草稿列表能显示：
  - [ ] 标题
  - [ ] 周范围
  - [ ] 中文状态
  - [ ] 更新时间
- [ ] 当前打开的 draft 会高亮
- [ ] 当前周的 draft 会显示“本周”标记

### 7. 左侧上下文面板

- [ ] 左侧完整显示五段：
  - [ ] 可汇报心得
  - [ ] 最近论文
  - [ ] 复现进展
  - [ ] 当前阻塞
  - [ ] 下周行动
- [ ] 每段都有空态文案与数量提示

### 8. 精确回跳

- [ ] 论文条目能跳到 `/search?paper_id=<paper_id>`
- [ ] 复现条目能跳到 `/reproduction?reproduction_id=<reproduction_id>`
- [ ] blocker 条目能跳到 `/reproduction?reproduction_id=<reproduction_id>`

## UI / 产品层面后续可完善点

下面这些不属于“当前版本验收失败项”，而是下一轮可继续打磨的方向。

### 1. 周报编辑器防误操作

- 切换草稿前增加未保存提醒

- 在 `context_hint` 之外增加更具体的关联对象标题
- 例如论文标题、复现摘要、心得摘要

### 6. 统一 badge / 状态视觉

- 草稿 / 已定稿 / 阻塞 / 进行中 / paper-only 等标签样式可进一步统一
