# Research Copilot 验收清单

本轮验收重点是：项目制主入口是否清晰、上下文是否连续、中文口径是否统一、数据来源是否解释清楚。

## 建议验收顺序

1. 项目首页与帮助抽屉
2. 项目工作台
3. 阅读器与旧页面深链
4. 周报与设置
5. 启动、测试与数据库说明

## 1. 项目首页与帮助抽屉

- [ ] 打开 `/` 或 `/projects` 时，首先看到的是项目首页，而不是旧模块仪表盘
- [ ] 顶栏主导航为：`项目 / 文库 / 周报 / 设置`
- [ ] 顶栏右侧按钮文案为中文，例如“搜索论文”“使用说明”
- [ ] 打开帮助抽屉后，内容围绕“项目工作台”主流程展开
- [ ] 帮助抽屉明确说明旧页面仍保留，但属于二级深链
- [ ] 帮助抽屉明确说明旧工作不会自动迁成项目

## 2. 项目首页

- [ ] 输入研究问题后可以正常创建项目并进入工作台
- [ ] 项目卡片能看到：标题、研究问题、状态、最近打开时间
- [ ] 项目卡片能看到：`论文 / 证据 / 成果物` 数量
- [ ] 项目卡片仍可编辑和删除
- [ ] 当没有项目时，空态会明确说明“这里只显示新项目对象”
- [ ] 当没有项目时，空态会提供跳转到 `文库 / 心得 / 复现 / 记忆 / 设置` 的入口

## 3. 项目工作台

- [ ] 项目内搜索、论文池、证据板、对比表、综述稿仍在同一工作台内完成
- [ ] 证据卡编辑会自动保存
- [ ] 对比表和综述稿自动保存后，刷新页面仍能保留
- [ ] 右侧任务面板能显示 AI 当前在做什么

## 4. 阅读器与旧页面深链

- [ ] 从项目进入阅读器时，会带上 `project_id`
- [ ] 阅读器顶部能返回项目工作台
- [ ] 阅读器里可以把段落直接加入当前项目证据板
- [ ] 从项目进入 `/search` 后，仍处于项目上下文
- [ ] 从项目进入 `/reflections` 后，默认看到项目范围内的心得
- [ ] 从项目进入 `/reproduction` 后，默认看到项目范围内的复现
- [ ] 从项目进入 `/memory` 后，默认看到项目范围内的记忆
- [ ] 从这些页面返回时，不会丢失原来的项目上下文

## 5. 周报与设置

- [ ] 周报仍然可正常生成和编辑
- [ ] 设置页能看到当前实际数据库路径
- [ ] 设置页会说明测试使用临时数据库，不污染开发库
- [ ] 设置页会说明旧工作不会自动迁成项目

## 6. 中文化与口径一致性

- [ ] 主导航、按钮、表单标签、空态、错误提示以中文为主
- [ ] 页面标题统一为“中文页面名 | Research Copilot”
- [ ] 不再出现“模块导航才是主入口”的旧表述
- [ ] “项目工作台是主入口，旧页面是二级深链”这一口径在首页、帮助、README、设置页一致
