# Research Copilot 产品审计与迭代纲要

本文档记录当前版本的产品定位、主流程和最近完成的收口批次。

## 当前产品定位

- 本地优先
- 单用户科研工作台
- 主入口是项目，不是聊天
- 英文论文内容 canonical，中文用于辅助理解和整理
- 目标是让用户围绕研究问题持续推进，而不是在模块之间来回找入口

## 当前主流程

1. 在 `项目` 页输入研究问题
2. 创建项目并进入项目工作台
3. 在项目内搜索、筛选、加入论文
4. 提取证据、生成对比表、起草综述
5. 必要时再深链进入阅读器 / 心得 / 复现 / 记忆
6. 在 `周报` 中整理本周输出

## 当前信息架构

- 一级主导航：`项目 / 文库 / 周报 / 设置`
- 二级深链入口：`搜索 / 心得 / 复现 / 记忆`
- 高级阅读器继续保留为深度工具，不承担首页引导职责

## 最近完成

### 2026-03-18：项目制工作台主重构

- 新增 `ResearchProject` 及其工作台数据结构
- 主入口切换到 `/projects`
- 项目内支持论文收集、证据板、对比表、综述稿
- 阅读器支持带 `project_id` 返回项目
- 旧页面支持项目上下文过滤

### 2026-03-18：自动保存、后台任务与 E2E

- 项目动作改为后台执行并支持流式进度
- 证据板支持拖拽排序与自动保存
- 对比表与综述稿支持自动保存和本地草稿镜像
- 接入 Playwright E2E 与临时数据库测试环境

### 2026-03-19：稳定性与中文化收口

- `GET /projects` 返回项目聚合计数，项目列表更容易识别
- 项目首页空态明确解释“旧工作不会自动迁成项目”
- 搜索 / 心得 / 复现 / 记忆页面统一项目上下文条与返回项目入口
- Memory 回跳直接进入阅读器，不再绕旧搜索跳板
- 顶栏、帮助抽屉、README、设置页与内部文档统一为项目制口径
- pytest 与 Playwright 默认使用临时数据库的说明进入产品与文档

## 当前阶段判断

当前项目已经进入“可日常使用的单人研究工作台”阶段，重点不再是扩很多新面，而是继续维持：

- 主流程清晰
- 数据来源可解释
- 上下文跳转不断裂
- 中文口径一致
- 自动保存和回归测试稳定

## 下一轮可选方向

- 阅读器段落高亮、定位和批注可视化再增强
- 翻译历史与快捷复用
- Memory 图谱视图增强
- 更强的项目内筛选、排序和批量操作
- 周报与项目输出之间更自然的联动
