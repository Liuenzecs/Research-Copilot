## 2026-03-29 拟定产品改进清单

### P0：先把主流程入口和阅读前置条件收口

- [ ] 应用启动后默认进入最近打开的项目工作台，而不是优先进入项目创建页
- [ ] 顶部“项目”导航默认回到最近打开项目；项目创建/项目列表入口下沉到项目工作台里的小按钮
- [ ] 加入项目后默认后台下载 PDF，并给出待下载/下载中/已下载/失败可重试状态
- [ ] 阅读器入口在 PDF 未下载完成时优先展示下载状态，而不是只给“当前尚未下载 PDF”阻断提示

### P1：把长列表改成稳定可扫读结构

- [ ] 项目工作台论文池增加分页，默认每页 20 篇，并与智能视图、阅读接续聚焦、排序联动
- [ ] 文库“我的文献”默认显示所有已下载论文，并增加分页
- [ ] 文库支持基础下载状态筛选：全部 / 已下载 / 下载失败或缺 PDF

### P2：补足文献元数据的中文可读性

- [ ] 论文入库时生成中文标题，同时保留英文原题
- [ ] 列表中优先显示中文标题，英文标题作为副标题保留
- [ ] 搜索同时匹配中英文标题
- [ ] 中文标题后续允许人工修正

### P3：重构周报工作区

- [ ] 周报页从“左上下文 + 右空草稿”改成“周期选择 + 周回顾摘要 + 草稿编辑”的三段式布局
- [ ] 周报摘要先突出：新入库论文、重点阅读进展、已提取证据、已记录心得、待处理事项
- [ ] 草稿编辑区改为更宽的主编辑区域，不再保持当前右侧窄栏

### 建议顺序

1. 启动入口与“项目”导航改造
2. 加入项目后默认后台下载 PDF
3. 项目工作台论文池分页
4. 文库“我的文献”分页与下载视图
5. 论文标题中英双语显示
6. 周报工作区重构

## 维护规则

1. 新一轮开发先明确是“流程收口”还是“能力扩展”
2. 继续以单用户、本地优先、Chinese-first 为默认标准
3. 涉及数据库与测试的变更，优先确认不会碰开发库
4. 每批完成后同步更新 README、帮助抽屉和本文件
